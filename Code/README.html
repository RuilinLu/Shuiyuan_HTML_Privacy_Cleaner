Shuiyuan HTML Privacy Cleaner V7

完全离线运行的 Windows 工具，用于清理由 SingleFile 保存的水源社区 / Discourse 单文件 HTML 快照。

V7 重点：全匿名模式在原始 SingleFile/Discourse HTML 上就地处理，不抽取帖子重写页面；保留论坛阅读布局、回复引用、图片、表情和话题统计区，同时去除保存者、公开用户、站点身份和隐藏预加载数据等隐私痕迹。

适用前提

  1. 在 Edge 浏览器中安装 SingleFile。
  2. 打开需要归档的水源社区页面。
  3. 用 SingleFile 保存当前页面，得到一个单文件 .html 快照。
  4. 用本工具清理这个 HTML，再分发清理后的副本。

SingleFile 参考仓库：https://github.com/RuilinLu/SingleFile

目前重点测试组合为 Edge + SingleFile + Windows x64 + win-x64 版本。win-x86 已在 x64 Windows 上做过命令行验证；win-arm64 已构建和校验架构，但未在 ARM64 真机验证。

界面说明

  1. 输入 HTML：选择 SingleFile 保存出来的原始 HTML。
  2. 输出 HTML：选择清理后文件的保存位置；留空时自动生成新文件名。
  3. 审核关键词：每行一个你额外担心的用户名、昵称、ID、头像编号、时间戳或路径片段。
  4. 模式：选择“仅清除当前保存者 / 登录者信息”或“全匿名模式”。
  5. 语言：支持简体中文、繁体中文和英文。
  6. 报告：支持易读摘要、标准报告和专业明细。
  7. 仅分析：只扫描输入文件，不写出新 HTML。
  8. 清理并审核：生成输出 HTML，并立即显示清理前、实际处理、清理后审核结果。
  9. 打开输出位置：打开输出文件所在文件夹。

运行时会显示真实百分比进度条。全匿名模式处理大文件会更慢，但不应空白卡死。

两种模式

仅清除当前保存者 / 登录者信息

移除水印、隐水印、SingleFile 保存时间、右上角账号入口、currentUser、我的帖子、浏览记录、未读计数、扩展残留和额外关键词等；公开帖子作者、站点名称和正文中的公开内容会保留。

全匿名模式

在上面基础上，继续匿名化所有公开用户，去除徽章/头衔/Flair、站点域名、站点名称、侧栏导航、隐藏预加载数据、原图 URL 和站点外链。同一个真实用户会稳定映射到同一套虚构头像、昵称和用户名。

V7 会保留 Discourse 的话题统计 / 参与者摘要行，例如“浏览量、赞、链接、用户 + 参与者头像”。这块会被匿名化，并强制横向一行显示，避免头像竖排错位。

命令行用法

ShuiyuanHtmlPrivacyCleaner.exe --clean input.html output.html --mode personal --report report.txt --report-style standard --language zh-CN --overwrite
ShuiyuanHtmlPrivacyCleaner.exe --clean input.html output_full.html --mode full --report report_full.txt --report-style standard --language zh-CN --overwrite

本地构建

需要 Windows 和 .NET SDK 9。

.\build.ps1

输出位于 dist/win-x64、dist/win-x86、dist/win-arm64，并生成 CHECKSUMS-SHA256.txt。

验收重点

  1. 清理后的 HTML 能在 Edge 中打开，不是空白页。
  2. 全匿名模式下，同一用户在帖子头部、头像、回复引用、@用户名、topic map 中使用同一套虚构身份。
  3. 话题统计 / 参与者摘要行仍然存在，并横向一行显示。
  4. 报告中的水印、currentUser、站点域名、公开用户标识、徽章/Flair、隐藏预加载数据、站点外链等审核项为 0。
  5. 源码仓库和发布包不包含私人测试 HTML、截图、报告或本机路径。

隐私说明

本工具针对 SingleFile 保存的水源/Discourse HTML 快照做静态清理。它会尽力去除已知水印、登录态、站点身份、用户身份和隐藏预加载数据，但不能数学证明任意未知隐写信息 100% 不存在。分发重要文件前，建议使用全匿名模式、额外关键词审核，并手动打开输出 HTML 复核可见内容。
